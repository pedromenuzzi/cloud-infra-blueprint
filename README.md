# Cloud Infra Blueprint

**Visual cloud infrastructure design synced bidirectionally with Terraform — in real time.**

A multi-tenant SaaS that pairs a Cloudcraft-style canvas with a Monaco HCL editor in a split view. Edit either side, see the other update in under 200 ms, with formatting, comments and block order preserved. Multi-cloud (AWS · Azure · GCP) since the MVP. Generates portable Terraform — no `terraform apply`, no lock-in.

## Why

Cloud teams live in two worlds: **diagrams** (Cloudcraft, Lucid, Excalidraw)
that go stale the moment infra moves, and **Terraform code** that is precise
but slow to explain visually. The tools that tried to bridge the two are either
closed (Brainboard), purely visual (Cloudcraft), or have been discontinued
(Terraform Visual Editor).

Cloud Infra Blueprint solves this with a **single source of truth** — a typed
intermediate representation (IR) of which the canvas and the HCL are just two
projections. Touch either side and the other updates instantly, without losing
formatting, comments, or block order.

## Highlights

- **Truly bidirectional.** Canvas ↔ HCL via canonical IR + minimal text patching that preserves trivia.
- **Multi-cloud day one.** AWS, Azure and GCP through a declarative, extensible catalog.
- **Runs in the browser.** HCL parser/emitter via WASM (`@cdktf/hcl2json`) in a Web Worker — backend stays thin.
- **Real-time collaboration.** Yjs CRDT keeps multiple users in sync without corrupting state (F3+).
- **Beautiful from the start.** Polished design system (Inter + JetBrains Mono, Tailwind tokens) with full light/dark/system themes.
- **Accessible by construction.** Axe-clean main routes, keyboard navigable, screen-reader friendly.
- **Offline first UX.** The web app falls back to bundled templates and demo projects when the API is unreachable.
- **No lock-in.** Exports straight Terraform (`main.tf`, `variables.tf`, `outputs.tf`, `providers.tf`) or pushes directly to GitHub/GitLab (F5).

## Quick start

```bash
# 1. Clone & install
git clone https://github.com/cloud-blueprint/cloud-blueprint.git
cd cloud-blueprint
pnpm install

# 2. Env
cp .env.example .env
# leave Clerk/GitHub OAuth empty for "dev no-auth" mode

# 3. Postgres + Redis
pnpm infra:up
pnpm db:migrate

# 4. Run web (:5173) + api (:3000)
pnpm dev
```

Open <http://localhost:5173>. The web app works **without the API** — it falls
back to the bundled templates and demo projects automatically. Spin up the API
when you want real persistence.

## The bidirectional contract

The IR is the only thing both sides can mutate. The canvas projects it as nodes
and edges; Monaco projects it as `.tf` text. Each edit produces a minimal patch
keyed off `trivia.rawTextRange` so comments and block order survive every
round-trip.

```
                  ┌──────────────────────────┐
                  │      canonical IR         │
                  │   (TypeScript ADT, in     │
                  │    memory + Yjs in F3+)   │
                  └─────────┬────────────────┘
                            │
          projection ↗      │      ↘ projection
                            │
       ┌────────────────┐   │   ┌──────────────────┐
       │  Canvas (RF)   │ ←─┴─→ │  Monaco (Yjs)    │
       └────────────────┘       └──────────────────┘
              ▲                          ▲
              │ drag/drop, connect       │ HCL keystrokes
              ▼                          ▼
         WebWorker                   WebWorker
         IR → HCL (patch)            HCL → IR (parse)
                @cdktf/hcl2json (WASM)
```

Full spec: [`docs/IR-SPEC.md`](docs/IR-SPEC.md).

## Quality gates

- **130+ unit tests** across IR, HCL, resources, templates, UI and routes.
- **Property-based tests** with `fast-check` ensure parser/emitter idempotency
  (1000 iterations on CI).
- **Round-trip ≥ 95% on 75+ Terraform fixtures** (currently 100%).
- **End-to-end** with Playwright covering landing → dashboard → templates →
  editor + theme switching + command palette.
- **Accessibility audits** with `@axe-core/playwright` on every main route.
- **CI** runs lint + typecheck + tests + build + E2E in parallel jobs with
  pnpm/turbo cache, on every PR.
